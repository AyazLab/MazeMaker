Dynamic Models are objects the user can interact with to complete the maze, in addition to making the maze seem more realistic. To add a Dynamic Object, select the "Dynamic" tool from the toolbar, and click in the Maze Editor to place an object there. Afterward, you must give it a Model for it to represent. When the Dynamic Model is assigned a model from the model collection, a "D" will appear in the circle representing the Dynamic Model in the Maze Editor, to show that it has been assigned a model. There must be items in the model collection in order to use Static or Dynamic Models.

Dynamic Models allow many types of user interaction. Aside from collisions, Dynamic Models can move, rotate, grow, shrink, change into another Model, or disappear (use the "Model Action" property for the object). They can be triggered by the user's proximity to the object (set "TriggerType" to "Proximity") or by completing a test (using readings from external sensors, select "Test" for "TriggerType"), or both. Growing and shrinking is done by changing the "ModelScale" and "End Scale" parameters. When the Model moves, rotates, or changes size, it does so with a smooth, linear transition.

    Other notes:
  • "HighlightStyle" only matters if "Criteria" is "Test" or "Proximity and Test"
  • "ActionTime", "End Point", "End Rotation", and "End Scale" only matter if "TriggerAction" is set to "Move/Rotate" or "Change Model"
  • When a Dynamic Model is moving, it cannot move the player, unless it moves downward under the player (in this case, gravity does the moving)
  • When a platform is moving upward under a player, the player can stay on by constantly jumping
